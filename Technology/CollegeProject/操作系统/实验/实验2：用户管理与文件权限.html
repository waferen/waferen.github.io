实验2：用户管理与文件权限

如何查看文件目录权限

在 Linux 系统中，包括通过 WSL（Windows Subsystem for Linux）运行的 Linux 发行版，你可以使用 ls 命令来查看文件和目录的权限。这里有几个常用的命令选项和方法来检查文件或目录的权限：

使用 ls -l 命令

ls -l 命令以长格式列出文件和目录信息，其中包括权限、所有者、组、大小、修改日期以及名称等信息。

ls -l [directory]

例如，如果你想查看当前目录下所有文件和子目录的详细信息，可以运行：

ls -l

输出示例：

-rw-r--r-- 1 user group 4096 Jan 29 14:33 file.txt
drwxr-xr-x 2 user group 4096 Jan 29 14:33 folder

在上述例子中，第一列 -rw-r--r-- 和 drwxr-xr-x 就是文件/目录的权限信息。这些信息分为四部分：

  • 第一个字符表示文件类型：- 表示普通文件，d 表示目录。
  • 接下来的三个字符 (rw-) 表示文件所有者的权限。
  • 再接下来的三个字符 (r--) 表示文件所属组的权限。
  • 最后的三个字符 (r--) 表示其他用户的权限。

权限字符的含义如下：

  • r (读) - 允许读取文件内容或列出目录内容。
  • w (写) - 允许修改文件内容或修改目录中的内容。
  • x (执行) - 允许执行文件或进入目录。

umask

umask 是 Unix 和类 Unix 系统（包括 Linux）中的一个命令和环境设置，用于确定新创建文件和目录的默认权限。umask 设置了一组权限位，这些权限位会被从文件或目录的默认最大权限中移除，从而得到实际的初始权限。

默认的最大权限

  • 对于普通文件，默认的最大权限是 666（即 -rw-rw-rw-），表示所有者、所属组和其他用户都有读写权限。
  • 对于目录，默认的最大权限是 777（即 drwxrwxrwx），表示所有者、所属组和其他用户都有读、写和执行权限。

umask 值

umask 的值是一个三位数的八进制数，每个数字代表不同用户的权限屏蔽：

  • 第一个数字：所有者的权限屏蔽
  • 第二个数字：所属组的权限屏蔽
  • 第三个数字：其他用户的权限屏蔽

在每个数字中：

  • 0 表示不屏蔽任何权限。
  • 1 表示屏蔽执行权限。
  • 2 表示屏蔽写权限。
  • 4 表示屏蔽读权限。
  • 7 表示屏蔽读、写和执行权限。

计算实际权限

实际权限 = 默认最大权限 - umask

例如，如果 umask 设置为 022，那么：

  • 对于文件，实际权限将是 666 - 022 = 644（即 -rw-r--r--）。
  • 对于目录，实际权限将是 777 - 022 = 755（即 drwxr-xr-x）。

查看当前的 umask

你可以在终端中输入以下命令来查看当前的 umask 设置：

umask

输出通常会显示为一个八进制数值，比如 0022 或 0002。

设置 umask

你可以通过在终端中运行 umask 命令并指定一个新的值来临时改变 umask。例如，要将 umask 设置为 027，可以这样做：

umask 027

这会使得新创建的文件具有 640 权限（-rw-r-----），而新创建的目录具有 750 权限（drwxr-x---）。

如果想永久更改 umask 设置，可以在 shell 的配置文件（如 .bashrc 或 .profile）中添加 umask 命令，这样每次登录时都会自动应用这个设置。

实验任务


首先，为系统添加四个用户，分别为他们设置密码，并在/home下创建一个共享目录work：

sudo useradd -d /home/alice -m alice
sudo useradd -d /home/bob -m bob
sudo useradd -d /home/john -m john
sudo useradd -d /home/mike -m mike

passwd alice
passwd bob
passwd john
passwd mike

mkdir /home/work

接着，创建一个用户组workgroup，将alice, bob, john加入到用户组workgroup，并将目录work的所有权改为alice和workgroup：

sudo groupadd workgroup
sudo usermod -aG workgroup alice
sudo usermod -aG workgroup bob
sudo usermod -aG workgroup john

chown alice.workgroup /home/work

现在，修改work目录的权限，使得属组内的用户对该目录具有完全访问权限，而属组外的用户对该目录没有访问权限：

chmod ug+rwX,o-rwx /home/work

尝试以bob的身份在work目录下创建文件bob.txt：

su - bob
cd /home/work
touch bob.txt

尝试以john的身份查看或修改bob.txt：

su - john
cd /home/work
cat bob.txt

尝试以mike的身份查看或修改bob.txt：

su - mike
cd /home/work
cat bob.txt

以mike的身份在其工作目录下创建mike.txt文件，并尝试将其移动到/home/work目录下：

su - mike
mkdir mike
chown mike.mike mike
touch mike.txt
mv mike.txt /home/work/

总结一下，这些命令演示了如何在Linux中管理用户和组权限，以及如何限制和授予对特定目录的访问权限。
