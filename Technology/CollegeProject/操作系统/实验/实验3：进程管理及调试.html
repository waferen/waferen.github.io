实验3：进程管理及调试

Linux中的进程管理

在 Linux 系统中，进程管理是一个非常重要的任务，它涉及到创建、监控、控制和终止进程。以下是一些常用的命令和工具，用于管理和操作 Linux 中的进程：

1. 查看进程

  • ps：显示当前终端的进程。

    ps

    你可以使用不同的选项来获取更详细的信息：

    ps -ef  # 显示所有用户的所有进程
ps -aux  # 显示所有进程的详细信息

  • top：实时显示系统中各个进程的状态。

    top

    top 命令会持续更新并显示系统的资源使用情况和进程状态。

  • htop：一个交互式的进程查看器，提供了比 top 更友好的界面。

    htop

    如果你没有安装 htop，可以使用包管理器进行安装：

    sudo apt-get install htop  # Debian/Ubuntu
sudo yum install htop      # CentOS/RHEL

  • pgrep：根据名称或其他属性查找进程 ID (PID)。

    pgrep process_name

  • pkill：根据名称或其他属性发送信号给进程。

    pkill process_name

2. 启动进程

  • 后台运行：在命令后面加上 & 可以将进程放到后台运行。

    command &

  • nohup：使进程在关闭终端后仍然继续运行。

    nohup command &

    输出会被重定向到 nohup.out 文件，除非你指定了其他输出文件。

  • screen 或 tmux：多窗口管理器，可以在断开连接后继续运行会话。

    screen
tmux

3. 控制进程

  • kill：向进程发送信号，最常用的是 SIGTERM（默认）和 SIGKILL。

    kill <PID>
kill -9 <PID>  # 发送 SIGKILL 信号

  • killall：根据名称杀死所有匹配的进程。

    killall process_name

  • nice 和 renice：调整进程的优先级。

    nice -n <priority> command  # 启动时设置优先级
renice <priority> -p <PID>  # 调整现有进程的优先级

4. 监控系统资源

  • free：显示系统内存使用情况。

    free -m  # 以 MB 为单位显示

  • df：显示磁盘空间使用情况。

    df -h  # 以人类可读的方式显示

  • iostat：显示 CPU 使用率和磁盘 I/O 统计信息。

    iostat -x 1  # 每秒更新一次

  • vmstat：报告虚拟内存统计信息。

    vmstat 1  # 每秒更新一次

5. 其他有用的命令

  • pstree：以树状图显示进程及其子进程。

    pstree

  • strace：跟踪系统调用和信号。

    strace -p <PID>

  • lsof：列出打开的文件和网络连接。

    lsof -p <PID>

  • netstat 或 ss：显示网络连接、路由表、接口统计等信息。

    netstat -tuln  # 显示所有监听端口
ss -tuln  # 替代 netstat 的现代工具

实验任务一


为了演示如何编写一个简单的 shell 程序并按照你的要求执行一系列操作，我将为你提供一个简单的 shell 脚本 badproc.sh，它会在后台无限循环创建和删除文件。注意，这个脚本只是为了演示目的，实际应用中应避免这样的行为。

1. 编写 badproc.sh

在文本编辑器中创建一个名为 badproc.sh 的文件，并输入以下内容：

#!/bin/bash

while true; do
    mkdir testdir
    sleep 1
    rmdir testdir
done

2. 为 badproc.sh 添加可执行权限

使用 chmod 命令为 badproc.sh 添加可执行权限：

chmod +x badproc.sh

3. 在后台执行 badproc.sh

在后台执行 badproc.sh：

./badproc.sh &

4. 利用 ps 查看其进程号

使用 ps 查找 badproc.sh 的进程号：

ps aux | grep badproc.sh

5. 利用 kill 命令杀死该进程

找到进程号后，使用 kill 命令杀死该进程：

kill <PID>

替换 <PID> 为实际的进程号。

整体流程

  1. 创建 badproc.sh：

    echo '#!/bin/bash' > badproc.sh
echo 'while true; do mkdir testdir; sleep 1; rmdir testdir; done' >> badproc.sh

  2. 增加可执行权限：

    chmod +x badproc.sh

  3. 在后台执行 badproc.sh：

    ./badproc.sh &

  4. 查看进程号：

    ps aux | grep badproc.sh

  5. 杀死进程：

    kill <PID>

  6. 删除目录和文件（如果有需要的话）：

    rm -r /path/to/files/directory
