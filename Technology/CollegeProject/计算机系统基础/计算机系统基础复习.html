字节与比特

字（Word）、字节（Byte）和比特（Bit）之间的换算关系以及字长的概念如下：

比特（Bit）

  • 比特是计算机中最小的信息单位。一个比特可以表示两种状态：0或1。

字节（Byte）

  • 1字节（Byte）= 8比特（Bits）。
  • 字节是计算机中用于表示数据的基本单位之一。通常一个字节可以表示256种不同的状态（2^8）。

字（Word）

  • 字（Word）是计算机中处理数据的基本单位，通常由多个字节组成。字的长度（字长）因计算机的体系结构而异。
  • 常见的字长有16位（2字节）、32位（4字节）和64位（8字节）。

字长（Word Length）

  • 字长是计算机一次能够处理的二进制数的位数。它决定了处理器的数据处理能力和计算精度。
  • 例如：
    • 一个16位字长的处理器可以一次处理16位数据。
    • 一个32位字长的处理器可以一次处理32位数据。
    • 一个64位字长的处理器可以一次处理64位数据。

换算关系总结

  • 1比特（Bit） = 1/8字节（Byte）。
  • 1字节（Byte） = 8比特（Bits）。
  • 字（Word）的长度是体系结构决定的，可以是2字节（16位）、4字节（32位）或8字节（64位）等。

大端方式和小端方式


对于大端方式，这里要存100 101 102 103 ，从左到右为字节从高到低，所以100是最高字节，103是最低字节，所以相当于把最高字节100存放在最低地址单元FF。
对于小端方式，这里要存103 102 101 100 ，从左到右为字节从高到低，所以103是最高字节，100是最低字节，所以相当于把最高字节103存放在最低地址单元FF。

寻址方式

IA-32常用指令


在AT&T汇编语言中，M和R通常用于指示操作数的类型和地址模式。具体来说：

M (Memory Operand)

  • M表示内存操作数。这意味着操作数位于内存中，可以是通过某种内存寻址方式访问的数据。
  • 内存操作数可以包括直接寻址、间接寻址、基址加偏移量等复杂的寻址方式。

例如：

movl $0x1234, 0x5678       # 将立即数0x1234移动到内存地址0x5678
movl %eax, (%ebx)          # 将寄存器eax的值移动到ebx寄存器所指向的内存位置

R (Register Operand)

  • R表示寄存器操作数。这意味着操作数是一个寄存器的内容。
  • 寄存器操作数可以是通用寄存器、段寄存器、控制寄存器等。

例如：

movl %eax, %ebx            # 将寄存器eax的值移动到寄存器ebx
addl %ecx, %edx            # 将寄存器ecx的值加到寄存器edx

具体使用场景

  1. 内存操作数 (M)

    movl $0x1234, 0x5678    # 将立即数0x1234移动到内存地址0x5678
movl %eax, 0x5678       # 将寄存器eax的值移动到内存地址0x5678
movl (%eax), %ebx       # 将eax寄存器指向的内存位置的值移动到寄存器ebx

  2. 寄存器操作数 (R)

    movl %eax, %ebx         # 将寄存器eax的值移动到寄存器ebx
addl %ecx, %edx         # 将寄存器ecx的值加到寄存器edx
subl %ebx, %eax         # 将寄存器ebx的值从寄存器eax中减去

即M操作时，是拿着算出来或者直接给的地址按图索骥，找到操作数。movl (%eax), %ebx，对于这种（）里的就表示找操作数。
而R操作代表了直接把寄存器里的机器数进行相减，并不访问内存。

leal和movl的区别

这两种操作对应了movl和leal，movl 0x18(%edx), %eax，是将0x18(%edx)的内容给eax，比如这个内存存的是数字4，就是将4这个值给eax。
leal则将 0x18(%edx)处的内容，比如是数字4，将4的地址给eax，其实就是不访问内存，只是将 0x18(%edx)算出的地址赋给后面的eax。

也就是说只有当用()这种寻址操作的时候，才是对内存里的数进行处理，其他都是对地址的操作。

简单的说就是 movl是把访问的内存内容赋值给寄存器， leal是將地址赋值给寄存器
举个例子
movl 0x18(%edx), %eax
leal 0x18(%edx), %eax
leal将 0x18(%edx)处的内容，比如是数字4，将4的地址给eax
movl则是将0x18(%edx)的内容直接给eax，比如是数字4，就是将4这个值给eax

add:
    # 函数 prologue
    pushl %ebp                # 保存旧的基址指针

    # 计算 x + y

    addl 12(%ebp), %eax       # 加上第二个参数 y

    popl %ebp                 # 恢复旧的基址指针
    ret                       # 返回结果在 eax 中

caller:
    # 函数 prologue
    pushl %ebp                # 保存旧的基址指针
    movl %esp, %ebp           # 设置新的基址指针

    # 设置局部变量 t1 和 t2
    subl $12, %esp            # 给局部变量分配空间
    movl $125, -4(%ebp)       # t1 = 125
    movl $80, -8(%ebp)        # t2 = 80

    # 调用 add 函数
    movl -4(%ebp), %eax       # 加载 t1 到 eax
    pushl %eax                # 压入参数 t1
    movl -8(%ebp), %eax       # 加载 t2 到 eax
    pushl %eax                # 压入参数 t2
    call add                  # 调用 add 函数
    addl $8, %esp             # 清理参数（恢复栈）

    # 保存结果到 sum
    movl %eax, -12(%ebp)      # sum = add(t1, t2)

    # 返回 sum
    movl -12(%ebp), %eax      # 将 sum 加载到 eax

    # 函数 epilogue
    movl %ebp, %esp           # 恢复栈指针
    popl %ebp                 # 恢复旧的基址指针
    ret                       # 返回结果在 eax 中

题目1

假设 x 为一个 int 型变量，请给出一个用来计算 x /32的值的函数div32。要求不能使用除法、乘法、模运算、比较运算、循环语句和条件语句，可以使用右移、加法以及任何按位运算。

要计算 x / 32 而不使用除法、乘法、模运算、比较运算、循环语句和条件语句，可以利用右移操作符来实现，因为整数除以 32 等同于向右移动 5 位（2^5 = 32）。不过，考虑到可能存在的负数情况，单纯的右移可能会导致问题，所以我们需要进行一些调整。

以下是计算 x / 32 的函数 div32：

int div32(int x) {
    // 先右移5位相当于除以32
    int result = x >> 5;

    // 为了处理负数情况，如果 x 是负数且对32有余数，需要将结果加1
    int bias = ((x & 31) != 0) && (x < 0);

    // 将 bias 转换为整数（0 或 1）并添加到 result
    return result + bias;
}

解释

  1. 右移5位：x >> 5，这相当于除以32。
  2. 处理负数的偏移量：
    • x & 31：这相当于计算 x 除以32的余数。
    • ((x & 31) != 0) && (x < 0)：这部分代码检查 x 是否为负数并且对32的余数不为零。如果条件为真，bias 为1，否则为0。
  3. 调整结果：将 bias 添加到 result 中。

通过这种方式，我们可以在不使用除法、乘法、模运算、比较运算、循环语句和条件语句的情况下实现 x / 32 的计算。

(x & 31) != 0这个为什么代表对32有余数

x & 31 实际上是获取 x 除以 32 后的余数，这是因为 31 的二进制表示是 11111（二进制的五个 1），只保留了 x 的最低五位。对于任何整数 x，x & 31 的值范围在 0 到 31 之间，这正好是 x 除以 32 的余数。

让我们举一些例子来说明：

例子 ：x = -34

  • -34 的二进制表示（假设32位机器）：11111111 11111111 11111111 11011110（注意这是补码表示）
  • 31 的二进制表示：00000000 00000000 00000000 00011111
  • -34 & 31 的结果：00000000 00000000 00000000 00011110（即 30）

所以，-34 除以 32 的余数是 30（-34 % 32 = 30）。

解释

  • x & 31：计算 x 除以 32 的余数。
  • (x & 31) != 0：判断是否存在余数。
  • x < 0：判断 x 是否为负数。
  • has_remainder & is_negative：如果 x 是负数并且有余数，则为 1，否则为 0。
  • 将这个值添加到 result 中，以调整负数除法的结果。

这样，我们可以在不使用除法、乘法、模运算、比较运算、循环语句和条件语句的情况下实现 x / 32 的计算，同时正确处理负数情况。

我们需要分析这个代码片段，并根据具体的条件和操作填写缺失的部分。该代码的目的是计算 (2^x) 的浮点表示，涉及到浮点数的位级表示。

代码分析

根据浮点数的 IEEE 754 标准，一个单精度浮点数的表示格式如下：

  • 1 位符号位
  • 8 位指数位
  • 23 位尾数位

要计算 (2^x) 的浮点表示，我们需要设置适当的指数和尾数位。

填写缺失部分

1. 值太小，返回 0.0

当 x 非常小（小于最小的可表示的指数），将返回 0.0。

if (x < -149) {
    exp = 0;
    frac = 0;
}

2. 返回非规格化结果

当 x在之间时，返回一个非规格化的结果。

else if (x < -126) {
    exp = 0;
    frac = 1 << (x + 149); // 计算尾数部分
}

3. 返回规格化结果

当 x在 之间时，返回一个规格化的结果。

else if (x <= 127) {
    exp = x + 127; // 将指数偏移 127
    frac = 0;
}

4. 值太大，返回 +∞

当 x 大于 127 时，返回正无穷大。

else {
    exp = 255;
    frac = 0;
}

完整的代码

float fpower2(int x)
{
    unsigned exp, frac, u;

    if (x < -149) { /* 值太小, 返回 0.0 */
        exp = 0;
        frac = 0;
    } else if (x < -126) { /* 返回非规格化结果 */
        exp = 0;
        frac = 1 << (x + 149);
    } else if (x <= 127) { /* 返回规格化结果 */
        exp = x + 127;
        frac = 0;
    } else { /* 值太大, 返回 +∞ */
        exp = 255;
        frac = 0;
    }

    u = (exp << 23) | frac;
    return u2f(u);
}

函数功能说明

fpower2 函数根据输入的整数 x 计算 (2^x) 的浮点表示。它将 x 转换为适当的指数和尾数部分，再通过 u2f 函数将其转换为浮点数并返回。根据 x 的值范围不同，函数处理了值太小、非规格化、规格化和值太大的情况。

关于尾数为什么是1 << (x + 149)

在IEEE 754标准的32位浮点数表示中，非规格化数用于表示非常接近于零的小数。对于单精度浮点数，非规格化数的指数部分全为0，且尾数部分用于表示实际的数值。

非规格化数的表示形式为：

其中，S 是符号位，0.f 是尾数位，2^{-126} 是固定的基数。这种表示法允许我们表示极小的数值，这些数值太小而无法用规格化形式表示。

在你的代码中，float fpower2(int x) 的目标是生成一个值为 2^x 的浮点数。为了理解为什么尾数是 1 << (x + 149)，我们需要了解非规格化数的范围和如何计算非规格化数。

解释 1 << (x + 149)

对于非规格化数，指数位为0，尾数部分决定了实际的数值。尾数部分表示的是一个小数点后的二进制值。例如，如果尾数部分是 0.00000000000000000000001，这对应的二进制数值是 2^{-23}。当你希望表示的数值是 2^x 且 x 在 -149 到 -126 之间时，你需要设置尾数部分。

由于尾数部分有23位，最小的非规格化数可以表示的是 2^{-149}，而最大的非规格化数可以表示的是 2^{-126}（仅使用尾数部分）。为了计算非规格化数的尾数部分，我们使用如下公式：

[ \text{尾数部分} = 1 << (x + 149) ]

这意味着我们希望将1左移 (x + 149) 位，以便在正确的位置设置尾数部分，使其表示 2^x。让我们举一个例子来说明这一点。

举例

假设 x = -148：

  1. x + 149 = 1。
  2. 1 << 1 即将 1 左移 1 位，结果是 0b10，对应于小数 0.00000000000000000000010，即 2^{-148}。

假设 x = -130：

  1. x + 149 = 19。
  2. 1 << 19 即将 1 左移 19 位，结果是 0b10000000000000000000，对应于小数 0.00000000000000000010000，即 2^{-130}。

通过将 1 左移 (x + 149) 位，我们能够准确地表示 2^x 在非规格化数的尾数部分。

32浮点数最大表示多少，最小表示多少

32位浮点数（即单精度浮点数，通常按照IEEE 754标准表示）由一个符号位、8个指数位和23个尾数位组成。

最大表示数

32位浮点数的最大表示数是当符号位为0（正数），指数位全为1且尾数位全为1时。具体计算如下：

  1. 符号位：0（正数）
  2. 指数位：最大非特殊值 254，即 11111110 （注意指数是有偏移的，偏移量为127，即偏移后的指数是 254 - 127 = 127）
  3. 尾数位：全为1，即 23位的 1

计算方法：
最大值最大值

最小表示数

32位浮点数的最小表示数可以分为以下几种情况：

正的最小非零值（次正规数）

次正规数时，指数位全为0，尾数位为1（最小的次正规数）。

  1. 符号位：0（正数）
  2. 指数位：全为0（此时指数值为 -126，偏移量为127，即偏移后的指数是 0 - 127 + 1 = -126）
  3. 尾数位：最小的次正规数为 0.00000000000000000000001（二进制，23位）

计算方法：
最小次正规数最小次正规数

正的最小正规数

最小的正规数时，指数位全为0，尾数位为0。

  1. 符号位：0（正数）
  2. 指数位：1（即 00000001，偏移量为127，即偏移后的指数是 1 - 127 = -126）
  3. 尾数位：全为0

计算方法：
最小正规数

概述

  • 最大正浮点数 ≈ (3.4 \times 10^{38})
  • 最小正非零次正规数 ≈ (1.4 \times 10^{-45})
  • 最小正正规数 ≈ (1.2 \times 10^{-38})

因此，32位浮点数的表示范围非常广，从极小的正数到极大的正数，可以表示极大的数字和极小的数字。
