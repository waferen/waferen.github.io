位运算

! ~ & ^ | + << >>这些符号在c语言中分别代表什么意思

  • !： 表示逻辑非操作符，当操作数为真时,result为0。否则，result 的值为 1。
  • ~： 表示按位非操作符，对操作数的每一位执行反转操作。
  • &： 表示按位与操作符，如果两个相应的二进制位都为1，那么结果为1，否则为0。
  • ^： 表示按位异或操作符，如果两个相应的二进制位值相同则为0，不同则为1。
  • |： 表示按位或操作符，如果两个相应的二进制位有一个为1，则结果为1，否则为0。
  • +： 表示加法操作符，用于将两个操作数相加。
  • <<： 表示左移操作符，将左操作数的二进制位向左移动指定的位数。在每次移动后，右侧空出的新位都填充为0。
  • >>： 表示右移操作符，将左操作数的二进制位向右移动指定的位数。在每次移动后，左侧空出的新位都填充为0或1（取决于数字的符号）。

!!(x)是什么意思

表达式!!(x)是一个经常使用的技巧，主要用于将任何非零整数转换为1，而保持0的值不变。

首先，为什么会这样呢？在C语言（以及许多其他语言）中，对于逻辑运算，任何非零的值都被视为真，而0被视为假。

! 是逻辑非操作符，它会将其操作数的逻辑值求反。也就是说，如果x非零（代表真），!x就会变为0（假）；如果x是0（假），!x则为1（真）。

然后再次使用!操作符，再次求反。因此，!!(x)，如果x非零（真），!!x就会变回为1（真）；如果x是0（假），!!x仍然为0（假）。这样，我们就实现了将所有非零值转换为1，而保持0不变的目的。

rotateLeft

代码
int rotateLeft(int x,int n){

    return (x << n) | (x >> (32 + (~n + 1))&((1<<n)+(~1+1)));//取反加一表示减法

}
(~1+(1<<n)+1)什么意思

这里确保了移动的位数超过整数的有效数据位部分（n位）会被屏蔽掉。(1 << n)得到的是将1左移n位之后的值，在此基础上加上1，然后再取反，得到的是一个从最高位开始，前32-n位为0，后n位为1的掩码（mask）。此掩码的作用在于只关注高位32-n位的数据，屏蔽掉x移动出范围的低n位数据。

parityCheck

按位异或0不变，按位异或1取反
代码
int parityCheck(int x){

    x ^= x >> 16;

    x ^= x >> 8;

    x ^= x >> 4;

    x ^= x >> 2;

    x ^= x >> 1;

    return x & 1;

}

通过不断的折半对比，让相对的一组1归零，如果最后一位剩下0,那么代表所有的1都被消掉，即为偶数个，如果最后一位剩下1，那么为奇数个1，与上1，即可取出最后一位。

absVal函数

代码
int absVal(int x){
    int y=x;
    y=y>>31;
    return ((~y)&x)+(y&(~x+1));
}

用((~y&x)+(y&(~x+1)）表示一个if判断y表示条件，&后面跟上不同条件对应的操作。
