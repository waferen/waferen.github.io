实验3：工厂场景的管理信息系统（MIS）设计

实验目的

1． 熟悉数据库设计的基本步骤和流程、熟练掌握ER模型设计工具。
2． 体会数据模型和数据模式在数据库设计中的作用，以及与数据库管理系统之间的关系。

实验平台与工具

  1. WSL2操作系统
  2. 数据库管理系统PostgreSQL
  3. ER设计工具：EZDML

需求分析

问题提出

假设某工厂有各种零件加工业务。每种零件有编号、名称、单价、类型等信息。工厂根据每一次工程的需要，从不同的供应商处批发零件。这些供应商和工厂长期合作，因此工厂有他们的名称、地址、电话等信息。每次批发都有一定的进货量。零件平时存放在各个仓库中，仓库有库房号、地址等信息。每个仓库都有一些职工，这些职工有职工号、姓名、性别、进厂时间以及不同的职称，每个仓库都设置了一个职工为组长，负责本仓库职工的管理。
进行需求分析是软件开发过程中的重要步骤，它帮助我们明确系统的目标、功能和需求。对于您描述的工厂零件加工业务，我们将从以下几个方面进行需求分析：

功能需求

1.1 零件管理

  • 零件信息管理：记录每种零件的编号、名称、单价、类型等信息。
  • 零件查询：根据零件编号、名称、类型等条件查询零件信息。
  • 零件维护：添加、修改、删除零件信息。

1.2 供应商管理

  • 供应商信息管理：记录供应商的编号、名称、地址、电话等信息。
  • 供应商查询：根据供应商编号、名称等条件查询供应商信息。
  • 供应商维护：添加、修改、删除供应商信息。

1.3 进货管理

  • 进货记录管理：记录每次进货的编号、零件编号、供应商编号、进货日期、进货数量等信息。
  • 进货查询：根据进货编号、零件编号、供应商编号、进货日期等条件查询进货记录。
  • 进货维护：添加、修改、删除进货记录。

1.4 仓库管理

  • 仓库信息管理：记录仓库的编号、地址等信息。
  • 仓库查询：根据仓库编号、地址等条件查询仓库信息。
  • 仓库维护：添加、修改、删除仓库信息。

1.5 职工管理

  • 职工信息管理：记录职工的编号、姓名、性别、进厂时间、职称等信息。
  • 职工查询：根据职工编号、姓名、职称等条件查询职工信息。
  • 职工维护：添加、修改、删除职工信息。
  • 组长管理：指定每个仓库的组长，负责本仓库职工的管理。

概念结构设计

概念结构设计步骤

  1. 识别实体：确定系统中需要管理的主要对象。
  2. 识别属性：确定每个实体的属性。
  3. 识别关系：确定实体之间的关系。
  4. 绘制实体关系图（ER图）：使用图形化的方式展示实体、属性和关系。

1. 识别实体

  • 零件 (Parts)
  • 供应商 (Suppliers)
  • 进货记录 (Purchases)
  • 仓库 (Warehouses)
  • 职工 (Employees)

2. 识别属性

零件 (Parts)

  • PartID (零件编号)
  • Name (名称)
  • Price (单价)
  • Type (类型)

供应商 (Suppliers)

  • SupplierID (供应商编号)
  • Name (名称)
  • Address (地址)
  • Phone (电话)

进货记录 (Purchases)

  • PurchaseID (进货编号)
  • PartID (零件编号)
  • SupplierID (供应商编号)
  • PurchaseDate (进货日期)
  • Quantity (进货数量)

仓库 (Warehouses)

  • WarehouseID (仓库编号)
  • Address (地址)
  • SupervisorID (组长编号)

职工 (Employees)

  • EmployeeID (职工编号)
  • Name (姓名)
  • Gender (性别)
  • JoinDate (进厂时间)
  • Title (职称)
  • WarehouseID (所属仓库编号)

3. 识别关系

零件与供应商

  • 进货记录：零件和供应商之间通过进货记录表建立关系。

仓库与职工

  • 职工管理：每个仓库有一个组长，负责管理该仓库的职工。

4. 绘制实体关系图（ER图）

解释

  1. 零件 (Parts)：

    • PartID：主键，唯一标识每个零件。
    • Name：零件的名称。
    • Price：零件的单价。
    • Type：零件的类型。

  2. 供应商 (Suppliers)：

    • SupplierID：主键，唯一标识每个供应商。
    • Name：供应商的名称。
    • Address：供应商的地址。
    • Phone：供应商的电话。

  3. 进货记录 (Purchases)：

    • PurchaseID：主键，唯一标识每次进货记录。
    • PartID：外键，引用零件表的主键。
    • SupplierID：外键，引用供应商表的主键。
    • PurchaseDate：进货日期。
    • Quantity：进货数量。

  4. 仓库 (Warehouses)：

    • WarehouseID：主键，唯一标识每个仓库。
    • Address：仓库的地址。
    • SupervisorID：外键，引用职工表的主键，表示仓库的组长。

  5. 职工 (Employees)：

    • EmployeeID：主键，唯一标识每个职工。
    • Name：职工的姓名。
    • Gender：职工的性别。
    • JoinDate：职工的进厂时间。
    • Title：职工的职称。
    • WarehouseID：外键，引用仓库表的主键，表示职工所属的仓库。

物理设计与实施

启动Postgresql服务

使用 service 命令

  1. 启动 PostgreSQL 服务：

    sudo service postgresql start

  2. 检查 PostgreSQL 服务状态：

    sudo service postgresql status

  3. 重启PostgreSQL服务:

    sudo service postgresql restart

创建数据库用户和数据库

  1. 切换到 PostgreSQL 用户：

    sudo -i -u postgres

  2. 创建数据库用户：

    createuser -P your_username
    • 按提示输入密码。

  3. 创建数据库：

    createdb -O your_username factory_management

  4. 退出 PostgreSQL 用户：

    exit

连接到数据库并创建表

  1. 连接到 PostgreSQL 数据库：

    psql -U your_username -d factory_management

    ps:如果失败则请配置pg_hba.conf文件，修改其中的peer权限验证为md5即可

  2. 创建表：

    -- 创建零件表
CREATE TABLE Parts (
    PartID SERIAL PRIMARY KEY,
    Name VARCHAR(100) NOT NULL,
    Price DECIMAL(10, 2) NOT NULL,
    Type VARCHAR(50)
);

-- 创建供应商表
CREATE TABLE Suppliers (
    SupplierID SERIAL PRIMARY KEY,
    Name VARCHAR(100) NOT NULL,
    Address TEXT,
    Phone VARCHAR(20)
);

-- 创建进货记录表
CREATE TABLE Purchases (
    PurchaseID SERIAL PRIMARY KEY,
    PartID INT REFERENCES Parts(PartID),
    SupplierID INT REFERENCES Suppliers(SupplierID),
    PurchaseDate DATE NOT NULL,
    Quantity INT NOT NULL
);

-- 创建仓库表
CREATE TABLE Warehouses (
    WarehouseID SERIAL PRIMARY KEY,
    Address TEXT,
    SupervisorID INT
);

-- 创建职工表
CREATE TABLE Employees (
    EmployeeID SERIAL PRIMARY KEY,
    Name VARCHAR(100) NOT NULL,
    Gender CHAR(1),
    JoinDate DATE NOT NULL,
    Title VARCHAR(50),
    WarehouseID INT REFERENCES Warehouses(WarehouseID)
);

-- 添加外键约束到仓库表
ALTER TABLE Warehouses
ADD CONSTRAINT fk_supervisor
FOREIGN KEY (SupervisorID) REFERENCES Employees(EmployeeID);
